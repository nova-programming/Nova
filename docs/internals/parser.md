# Nova Parser Internals (`stdlib/parser.nv`)

The `parser.nv` module is responsible for syntax analysis. It consumes the linear stream of `Token` objects generated by the tokenizer and builds a hierarchical Abstract Syntax Tree (AST) representing the semantic structure of the code.

## State Management

The parsing process uses a `ParserState` struct containing:
* `tokens`: The list of tokens produced by the tokenizer.
* `pos`: The current index in the token list.
* `length`: The total number of tokens.

The parser employs a recursive descent architecture. It primarily advances through tokens using utility functions like `peek()`, `advance()`, `expect()`, and `match()`.

## Node Hierarchy

Every element in the AST is represented by an `AstNode`. The `AstNode` uses a generalized struct capable of holding properties for multiple specific grammar components. Common properties include:
* `kind`: A string indicating the node type (e.g., `"Program"`, `"FunctionDef"`, `"IfElse"`, `"BinOp"`, `"Variable"`).
* `val_str` / `val_int`: Primitive values (e.g., function names, integer literals).
* `body` / `if_body` / `else_body`: Lists of child `AstNode`s for block statements.
* `left` / `right`: Child nodes for binary operations.

## Recursive Descent Flow

1. **Top-Level Parsing (`parse(state)`):**
   The entry point loops until the end of tokens, parsing top-level definitions like `def` (functions), `struct` (structs), or loose statements.

2. **Statement Parsing (`parse_stmt(state)`):**
   Dispatches based on the current token.
   - `def`: Invokes `parse_function()`.
   - `if`: Invokes `parse_if_else()`.
   - `while`: Invokes `parse_while()`.
   - `@raw`: Invokes `parse_raw_block()`.
   - Otherwise: Attempts to parse an expression (`parse_expr()`) followed by an optional assignment (`=`).

3. **Expression Parsing (`parse_expr(state)` / `parse_binop(...)`):**
   Handles operator precedence. It parses a primary expression (e.g., variable, literal) and then looks ahead for binary operators like `+`, `-`, `==`, `!=`, `<`, `>`. If found, it constructs a `BinOp` node recursively folding left-to-right.

4. **Primary Expressions (`parse_primary(state)`):**
   Handles the smallest atomic units:
   - Integers and Strings.
   - Variables (identifiers).
   - Parenthesized expressions `( expr )`.
   - List literals `[ expr, expr ]`.
   - Handles suffix operations like function calls `func()`, property accesses `.prop`, or array indexing `[idx]`.

## Type Declarations
The parser optionally consumes type annotations (e.g., `def foo() -> int { ... }`) to validate signatures and assist with potential native typing rules.

## Special Constructs
* **Raw Blocks:** `@raw { ... }` parses standard statements but wraps them in a `"RawBlock"` kind. The native compiler handles these blocks distinctly, primarily by altering how variables are allocated or scoped.
